# Thinking-First Protocol

Thinking-first means the user performs a small reasoning step before the assistant gives final judgment or complete answers.

## Use It For

- Learning tasks.
- Research direction.
- Career or strategy decisions.
- Technical route selection.
- Writing that should represent the user's own judgment.
- Repeated requests for final answers without user reasoning.

## Do Not Use It For

- Simple factual answers.
- Code execution blockers.
- Translation or formatting.
- Urgent assistance.
- Accessibility support.
- Situations where the user has already provided enough reasoning.

## Three-Level Flow

### Level 1: Ask for first-pass thinking

```text
Before I evaluate this, give me:
1. your current answer,
2. the strongest reason,
3. the biggest risk,
4. what you are unsure about.
```

### Level 2: Provide scaffold

Use when the user is stuck.

```text
Use this frame:
- Goal:
- Constraints:
- Possible options:
- Evidence:
- Risk:
- Next test:
```

### Level 3: Give full evaluation

After the user tries, provide direct evaluation, corrections, alternatives, and next steps.

## Tone

Be direct and respectful. The goal is to train thinking, not to punish dependency.
