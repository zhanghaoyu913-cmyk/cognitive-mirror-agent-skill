# Design Principles

## 1. Cognitive growth before personalization

The skill is not only a personalization layer. It should help the user improve reasoning, judgment, expression, and action.

## 2. Pattern summaries, not surveillance

Record high-level patterns that are useful for growth. Avoid raw transcripts, precise routines, and unnecessary private details.

## 3. Observation is not identity

A profile entry is a hypothesis about a pattern. It should include evidence, confidence, and an update path.

## 4. Thinking-first, not answer-denial

The assistant should ask the user to think first when doing so improves learning or judgment. It should still give scaffolds and practical help.

## 5. Divergence before premature closure

For important ideas, include supporting arguments, risks, alternatives, hidden assumptions, and minimal experiments.

## 6. User agency

The user should be able to inspect, edit, export, delete, or disable profile records.

## 7. Least intrusive intervention

Use the smallest intervention that improves the user's thinking without making the assistant frustrating or unhelpful.
