# Privacy Policy Draft

This document is a project-level privacy template. Adapt it before using the skill in a production product.

## Purpose

Cognitive Mirror is designed to help users reflect on thinking patterns, AI usage habits, and growth opportunities. It is not designed for surveillance, behavioral tracking, advertising, or sensitive profiling.

## Data to Record

The skill may record high-level summaries such as:

- Preferred explanation style.
- Learning and reasoning patterns.
- Recurring blockers.
- AI usage habits.
- User-stated growth goals.
- Assistant adaptation suggestions.

## Data to Avoid

The skill should not record:

- Raw private conversations.
- Passwords, tokens, credentials, or account identifiers.
- Precise location.
- Health or mental-health assumptions.
- Sensitive identity attributes.
- Detailed daily routines.
- Private information about third parties.

## User Rights

Any implementation with persistent storage should provide:

- Review.
- Edit.
- Export.
- Delete.
- Disable profiling.

## Retention

Profile entries should include timestamps and confidence levels. Stale entries should be reviewed or removed.

## Consent

Durable memory should be opt-in or clearly disclosed. Sensitive profile updates should require explicit user confirmation.
